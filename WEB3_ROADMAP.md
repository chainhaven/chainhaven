# Chain Haven Web3 Roadmap

## Positioning

Chain Haven is a Web3 builder identity focused on secure onchain systems.

Main image:

```text
Trusted Web3 builder. Security-minded. Practical. Long-term.
```

Avoid:

```text
Hype, fake alpha, gambling energy, empty crypto flexing.
```

## Core Path

Start with this order:

1. Git and GitHub
2. JavaScript / TypeScript
3. Solidity
4. Foundry
5. Smart contract testing
6. Wallet integration
7. React / Next.js dApps
8. Onchain data
9. DeFi basics
10. Security and audits

## Portfolio Projects

Build these in public:

1. `solidity-vault`

   A simple ETH vault contract with deposit, withdraw, events, tests, and security notes.

2. `erc20-launchpad`

   A clean ERC20 token deployer with tests and a basic frontend.

3. `wallet-dashboard`

   A dApp that connects a wallet and shows ETH balance, token balances, and recent activity.

4. `defi-risk-notes`

   A research repo explaining DeFi risks in simple language: liquidation, oracle risk, admin keys, smart contract bugs, and bridge risk.

5. `onchain-tools`

   Small scripts for reading contract data, checking token approvals, and tracking wallet activity.

## GitHub Style

Every project should include:

- Clear `README.md`
- What the project does
- Tech stack
- How to run it
- Screenshots if there is a frontend
- Security notes
- What you learned

Commit style:

```text
feat: add vault deposit function
test: cover withdraw edge cases
docs: explain contract security assumptions
fix: prevent zero amount deposits
```
